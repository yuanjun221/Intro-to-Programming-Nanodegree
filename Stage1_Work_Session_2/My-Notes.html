Tree Structure

HTML

HTML is the language of the web, it controls the "structure" of a web page
The browser turns HTML tags into elements that form a tree(DOM)
Each html tag creat an element in DOM that the browser use to display the page

basic word: tag
element: <tag>content</tag>

the content can be empty, text or in other element
each tag has a name, or type: defines what kind of element will be created
each tag has attributes with values:
<h1 class="article">content</h1>
h1: tag class:attribute article:value of attribute

DOM (Document Object Model)

It's a standard convention for representing and interacting with elements in HTML.
The tree in developer tools is the DOM tree that browser build from a html document.

CSS (Cascading Style Sheets)

a language controls the "style" of a page (how it looks)

Box

everything in the web page is actually a box
use <div>content</div> to creat a box

CSS Vocabulary

CSS : Cascading Style Sheets

With CSS the most specific rule is applied to every element

Cascading

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

Style

Style can refer: the HTML element
the HTML attribute
the general look of a web page
the verb e.g. "The structure is all done but now I have to style it."

Rule

a line of CSS code describing the value that a certain attribute should take.

Property

The property you want to change.

Value

The value that you want to assign to the attribute.

Selector

select an HTML tag, that rule will apply to all tags on the page:
e.g.
p {
...
}
will apply style to all paragraphs on a page

select elements by class:
e.g.
.description {
color:red;
}
will apply style to all elements with class "description"

syntax:
inside brackets, write the style in the from [attribute colon value]

Class

A class refers to a group of elements that can be styled together. Class names should not contain periods or any other punctuation marks such as class="1.1"

ID

ID's are unique identifiers that uniquely identifies an element in HTML.

Defining Style and Order Matters

Styles can be defined in different places and are applied in the following order, with definitions further down the list overwriting previous definitions:

* the default style of a browser
* stylesheet in a separate file
* stylesheet inside HTML
* inline style in an element
